Fetch API: Cách 'giao tiếp' với Server bằng JavaScript

Làm thế nào để một trang web hiển thị dữ liệu mới mà không cần tải lại trang? Bí mật nằm ở API. Bài viết này sẽ hướng dẫn bạn cách dùng Fetch API, công cụ tiêu chuẩn để lấy dữ liệu từ server.
October 12, 2025 · Hữu Khang

Giới thiệu: API - Người phục vụ của thế giới Web#

Hãy tưởng tượng trang web của bạn là một thực khách trong nhà hàng. Bạn (người dùng) muốn gọi một món ăn (dữ liệu). Bạn không thể tự mình chạy vào bếp (server) để lấy, mà bạn cần một người phục vụ.

API (Application Programming Interface) chính là “người phục vụ” đó. Nó là một cổng giao tiếp cho phép trang web của bạn gửi yêu cầu (“order món”) đến server và nhận lại dữ liệu (“món ăn đã được chuẩn bị”). Dữ liệu này thường được trả về dưới định dạng JSON.

Trong JavaScript hiện đại, công cụ tiêu chuẩn để làm việc này chính là Fetch API.

1. Cú pháp cơ bản của fetch()#

fetch() là một hàm có sẵn trong trình duyệt, giúp chúng ta gửi đi một yêu cầu mạng.

  • Đầu vào: Nhận vào một đường dẫn (URL) của API.
  • Đầu ra: Nó trả về một Promise. Điều này có nghĩa là chúng ta có thể dùng .then() và .catch() (hoặc async/await) để xử lý kết quả, giống như bài viết trước!

Quá trình fetch dữ liệu thường gồm 2 bước:

  1. Gọi fetch(url) và nhận về một đối tượng Response. Đối tượng này chưa phải là dữ liệu bạn cần, mà chỉ là thông tin chung về phản hồi từ server.
  2. Gọi phương thức .json() trên đối tượng Response đó để chuyển đổi dữ liệu thành một đối tượng JavaScript mà chúng ta có thể sử dụng.

Ví dụ Code dùng Promise:#

Để thực hành, chúng ta sẽ dùng API miễn phí từ JSONPlaceholder, một trang web tuyệt vời để test API.

// Đường dẫn API chúng ta muốn lấy dữ liệu
const apiUrl = '[https://jsonplaceholder.typicode.com/todos/1](https://jsonplaceholder.typicode.com/todos/1)';

console.log("Bắt đầu gọi API...");

fetch(apiUrl)
  .then(response => {
    // Bước 1: Kiểm tra xem server có trả về thành công không (status 200)
    if (!response.ok) {
      throw new Error('Network response was not ok ' + response.statusText);
    }
    // Chuyển đổi response thành JSON, bước này cũng trả về một Promise
    return response.json();
  })
  .then(data => {
    // Bước 2: Nhận được dữ liệu JSON cuối cùng
    console.log("Đã nhận được dữ liệu:");
    console.log(data);
    // Kết quả: { userId: 1, id: 1, title: 'delectus aut autem', completed: false }
  })
  .catch(error => {
    // Bắt lỗi nếu có sự cố mạng hoặc server
    console.error('Đã có lỗi xảy ra khi fetch data:', error);
  });

console.log("Code vẫn chạy tiếp mà không cần chờ API xong!");

2. Cách viết hiện đại hơn với Async/Await#

Như chúng ta đã học, async/await giúp code bất đồng bộ trông gọn gàng và dễ đọc hơn rất nhiều.

Ví dụ Code dùng Async/Await:#

// Đường dẫn API
const apiUrl = '[https://jsonplaceholder.typicode.com/todos/1](https://jsonplaceholder.typicode.com/todos/1)';

// Phải đặt trong một hàm async
async function getTodo() {
  console.log("Bắt đầu gọi API với async/await...");
  try {
    // "await" sẽ tạm dừng hàm cho đến khi fetch hoàn thành
    const response = await fetch(apiUrl);

    // Kiểm tra lỗi
    if (!response.ok) {
      throw new Error('Network response was not ok');
    }

    // "await" tiếp để chờ quá trình chuyển đổi JSON xong
    const data = await response.json();

    console.log("Đã nhận được dữ liệu:");
    console.log(data);
  } catch (error) {
    // Bắt lỗi ở một nơi duy nhất
    console.error('Đã có lỗi xảy ra:', error);
  }
}

// Gọi hàm để thực thi
getTodo();
