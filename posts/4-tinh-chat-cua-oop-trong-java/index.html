OOP: 4 Tính Chất Cốt Lõi Của Lập Trình Hướng Đối Tượng trong Java

Bài viết này sẽ giải thích chi tiết về 4 tính chất cốt lõi của OOP: Tính đóng gói, Kế thừa, Đa hình và Trừu tượng qua các ví dụ Java dễ hiểu.
October 10, 2025 · Hữu Khang

Lời mở đầu#

Lập trình Hướng đối tượng (OOP) là một mô hình lập trình cực kỳ quan trọng và là nền tảng của ngôn ngữ Java. Nó giúp chúng ta tổ chức code một cách logic, dễ bảo trì và tái sử dụng. Trái tim của OOP nằm ở 4 tính chất cốt lõi: Tính đóng gói, Kế thừa, Đa hình, và Trừu tượng. Trong bài viết này, chúng ta sẽ cùng nhau tìm hiểu chi tiết về từng tính chất nhé.

1. Tính Đóng Gói (Encapsulation)#

  • Giải thích: Viết 2-3 dòng giải thích Tính đóng gói là gì. Hãy nghĩ về nó như một viên thuốc con nhộng, che giấu dữ liệu và các phương thức xử lý bên trong, chỉ cho phép tương tác qua các giao diện công khai. Mục đích là để bảo vệ dữ liệu.
  • Cách triển khai trong Java: Nói về việc sử dụng access modifier private cho các thuộc tính và public cho các phương thức getter và setter.
  • Ví dụ Code: 
    // Ví dụ về lớp Student
public class Student {
    private String name; // Dữ liệu được đóng gói
    private int age;

    // Cung cấp phương thức public để truy cập
    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        if (age > 0) { // Có thể kiểm tra dữ liệu đầu vào
            this.age = age;
        }
    }
}

2. Tính Kế Thừa (Inheritance)#

  • Giải thích: Kế thừa cho phép một lớp (lớp con) thừa hưởng lại các thuộc tính và phương thức của một lớp khác (lớp cha). Nó thể hiện mối quan hệ “IS-A” (là một). Mục đích là để tái sử dụng code.
  • Cách triển khai trong Java: Nói về từ khóa extends và super.
  • Ví dụ Code: 
    // Lớp cha
class Animal {
    void eat() {
        System.out.println("This animal eats food.");
    }
}

// Lớp con kế thừa từ Animal
class Dog extends Animal {
    void bark() {
        System.out.println("The dog barks.");
    }
}

3. Tính Đa Hình (Polymorphism)#

  • Giải thích: Đa hình có nghĩa là “một đối tượng có nhiều hình thái”. Nó cho phép chúng ta thực hiện một hành động theo nhiều cách khác nhau. Dạng phổ biến nhất là Ghi đè phương thức (Method Overriding).
  • Cách triển khai trong Java: Nói về annotation @Override.
  • Ví dụ Code: 
    class Animal {
    public void makeSound() {
        System.out.println("Some generic animal sound");
    }
}

class Cat extends Animal {
    @Override // Ghi đè phương thức của lớp cha
    public void makeSound() {
        System.out.println("Meow meow");
    }
}

4. Tính Trừu Tượng (Abstraction)#

  • Giải thích: Tính trừu tượng giúp che giấu đi các chi tiết triển khai phức tạp và chỉ hiển thị các chức năng cần thiết cho người dùng. Giống như bạn lái xe hơi mà không cần biết động cơ bên trong hoạt động ra sao.
  • Cách triển khai trong Java: Nói về lớp trừu tượng (abstract class) và giao diện (interface).
  • Ví dụ Code: 
    // Lớp trừu tượng
abstract class Shape {
    // Phương thức trừu tượng, không có phần thân
    abstract void draw();
}

class Circle extends Shape {
    void draw() {
        System.out.println("Drawing a circle.");
    }
}

Kết luận#

Tóm tắt lại sức mạnh của 4 tính chất OOP khi kết hợp với nhau. Chúng giúp code Java trở nên linh hoạt, an toàn và có cấu trúc tốt hơn như thế nào.
