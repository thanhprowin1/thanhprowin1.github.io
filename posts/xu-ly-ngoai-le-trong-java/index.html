OOP: Xử lý ngoại lệ (Exception Handling) trong Java: Tấm khiên bảo vệ chương trình của bạn

Chương trình bị 'sập' bất ngờ là nỗi ác mộng của mọi developer. Bài viết này sẽ hướng dẫn bạn từ A-Z cách sử dụng Exception Handling trong Java để xử lý lỗi một cách duyên dáng và chuyên nghiệp.
October 12, 2025 · Hữu Khang

Giới thiệu#

Khi lập trình, chúng ta không thể nào lường trước được mọi tình huống. Điều gì sẽ xảy ra nếu người dùng nhập vào một chữ thay vì một số? Hoặc nếu chương trình cố gắng đọc một file không tồn tại? Nếu không được xử lý, những “tai nạn” này, hay còn gọi là ngoại lệ (Exception), sẽ khiến chương trình của bạn bị dừng đột ngột (crash).

Exception Handling (Xử lý ngoại lệ) chính là tấm khiên cho phép chúng ta bắt lấy các lỗi này, xử lý chúng một cách êm đẹp, và giữ cho chương trình tiếp tục hoạt động.

1. Hệ thống phân cấp Exception trong Java#

Trong Java, tất cả các ngoại lệ đều kế thừa từ lớp Throwable. Hệ thống này được chia làm hai nhánh chính:

  • Error: Đây là những lỗi nghiêm trọng xảy ra bên trong máy ảo Java (JVM), thường liên quan đến tài nguyên hệ thống (ví dụ: OutOfMemoryError). Chúng ta gần như không thể và cũng không nên cố gắng bắt (catch) các lỗi này.

  • Exception: Đây là những lỗi mà chương trình của chúng ta có thể lường trước và xử lý được. Nhánh này lại được chia làm hai loại nhỏ hơn:

    • Checked Exception: Là các ngoại lệ mà trình biên dịch Java bắt buộc bạn phải xử lý. Ví dụ: IOException (khi làm việc với file), SQLException (khi làm việc với database).
    • Unchecked Exception (Runtime Exception): Là các ngoại lệ xảy ra lúc chương trình chạy (runtime), thường do lỗi logic của lập trình viên. Trình biên dịch không bắt buộc bạn phải xử lý chúng. Ví dụ: NullPointerException, ArrayIndexOutOfBoundsException.

2. Các từ khóa “ma thuật”: try, catch, finally, throw, throws#

Đây là 5 từ khóa chính tạo nên cơ chế xử lý ngoại lệ trong Java.

a. Khối try-catch#

Đây là cấu trúc cơ bản nhất.

  • try: Đặt đoạn code có nguy cơ gây ra ngoại lệ vào trong khối này.
  • catch: Nếu một ngoại lệ xảy ra trong khối try, chương trình sẽ nhảy ngay đến khối catch tương ứng để xử lý mà không thực thi phần còn lại của khối try.
public class TryCatchExample {
    public static void main(String[] args) {
        try {
            // Đoạn code có nguy cơ gây lỗi
            int result = 10 / 0; // Lỗi chia cho 0 (ArithmeticException)
            System.out.println("Kết quả là: " + result); // Dòng này sẽ không bao giờ được thực thi
        } catch (ArithmeticException e) {
            // Xử lý lỗi
            System.out.println("Lỗi rồi bạn ơi: Không thể chia một số cho 0.");
            // e.printStackTrace(); // Dùng để in ra chi tiết lỗi
        }
        System.out.println("Chương trình vẫn tiếp tục chạy!");
    }
}

b. Khối finally#

  • finally: Dù có ngoại lệ xảy ra hay không, khối finally luôn luôn được thực thi. Nó thường được dùng để dọn dẹp tài nguyên, ví dụ như đóng file hoặc đóng kết nối database.
import java.io.FileReader;
import java.io.IOException;

public class FinallyExample {
    public static void main(String[] args) {
        FileReader reader = null;
        try {
            reader = new FileReader("non_existent_file.txt");
        } catch (IOException e) {
            System.out.println("Lỗi: Không tìm thấy file!");
        } finally {
            // Dù có lỗi hay không, luôn cố gắng đóng file
            System.out.println("Khối finally luôn được gọi.");
            if (reader != null) {
                try {
                    reader.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}

c. throw và throws#

  • throw: Dùng để chủ động ném ra một ngoại lệ. Bạn có thể tự tạo và ném ra một ngoại lệ của riêng mình.
  • throws: Được dùng trong khai báo phương thức để “thông báo” rằng phương thức này có khả năng ném ra một (hoặc nhiều) Checked Exception. Ai gọi phương thức này sẽ có trách nhiệm phải xử lý các ngoại lệ đó.
public class ThrowThrowsExample {

    // Phương thức này "thông báo" nó có thể ném ra một Exception
    public static void validateAge(int age) throws Exception {
        if (age < 18) {
            // Chủ động ném ra một ngoại lệ mới
            throw new Exception("Tuổi không hợp lệ, bạn phải đủ 18 tuổi.");
        }
        System.out.println("Tuổi hợp lệ.");
    }

    public static void main(String[] args) {
        try {
            // Vì validateAge có `throws Exception`, ta phải gọi nó trong khối try-catch
            validateAge(15);
        } catch (Exception e) {
            System.out.println("Bắt được ngoại lệ: " + e.getMessage());
        }
    }
}
